- `Super_Mechanic\Helpers\PDF_Service::can_generate_pdf()` -> FAIL
- reason: the shared invoice/quote `PDF_Service` does not load embedded TCPDF in a fresh request

  - `sm_pdf_engine_unavailable`

Contract impact:
- the exact code-level fix belongs in `includes/helpers/class-pdf-service.php`
- that file is outside the 56P11-C allowed files list
- no code fix was applied in this retake

Resolution:

- reporting reuses the shared loader path

Root cause confirmed:
- `Report_PDF_Service` loaded bundled TCPDF, but shared `PDF_Service` did not.
- Invoice and quote exports use shared `PDF_Service`, so they failed in fresh requests until another service loaded `TCPDF`.

Fix applied:
- `PDF_Service::can_generate_pdf()` now loads bundled TCPDF before checking engine classes.
- `PDF_Service::generate_pdf_binary()` now also attempts the bundled loader before selecting an engine.
- `Report_PDF_Service` now delegates engine availability to the parent shared loader.
