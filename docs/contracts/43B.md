PHASE: 43B
NAME: Ejecución semi-automática (confirmación requerida)

==================================================
OBJETIVO
==================================================

Permitir que reglas activadas preparen una acción ejecutable
y que el usuario la confirme explícitamente antes de ejecutarla.

==================================================
SCOPE
==================================================

- preparar ejecución semi-automática desde reglas activadas
- mostrar acción lista para confirmar
- ejecutar solo tras confirmación manual
- reutilizar handlers ya existentes

==================================================
OUT OF SCOPE
==================================================

- autoejecución sin confirmación
- cron
- triggers automáticos
- nuevas tablas
- persistencia compleja
- rollback automático
- tocar CRM Pipeline core

==================================================
ALLOWED FILES
==================================================

- includes/automation/class-operational-rules-service.php
- includes/dashboard/class-workload-service.php
- includes/dashboard/class-admin-dashboard-controller.php
- assets/css/admin.css
- docs/contracts/43B.md
- docs/contracts/validation/43B-validation.md

==================================================
FORBIDDEN FILES
==================================================

- includes/modules/*
- schema/database
- includes/crm/* salvo reutilización indirecta ya existente
- includes/processes/*
- includes/appointments/*
- invoices/payments core

==================================================
OBJETIVO FUNCIONAL
==================================================

Cuando una regla esté triggered y exista acción segura soportada,
mostrar:

- preview de la acción
- impacto
- cantidad de ítems afectados
- botón `Confirm and Run`

Acciones iniciales soportadas:

1. overdue_tasks_cleanup
   - acción semi-automática:
     - `bulk_resolve`

2. critical_saturation_rebalance
   - acción semi-automática:
     - `bulk_reassign`
   - solo si existe propuesta ejecutable real

3. multi_critical_alert
   - no mutación directa
   - solo navegación / open_center
   - no confirm action destructive

==================================================
CRITERIO DE IMPLEMENTACIÓN
==================================================

- reutilizar 42C y 43A
- no duplicar lógica de ejecución
- no recalcular reglas fuera del servicio
- confirmación obligatoria antes de ejecutar
- no ejecutar nada en render GET
