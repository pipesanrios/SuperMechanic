## Scope
- Restore embedded PDF engine detection/loading for reporting PDF exports.
- Audit embedded TCPDF candidates:
  - `includes/libs/pdf/tcpdf/tcpdf.php`
  - `vendor/tecnickcom/tcpdf/tcpdf.php`
  - `vendor/autoload.php`
- Ensure reporting PDF service detects an embedded engine when physically present.
- Preserve reporting export flow, calculations, data mapping, filenames and response behavior.

## Allowed Files
- `includes/reporting/class-report-pdf-service.php`
- `includes/pdf/*`
- `includes/libs/pdf/*`

- `docs/CURRENT_STATE.md`
- `docs/QA_REPORT.md`

## Forbidden
- No external plugin/library dependency requirement for users.
- No reporting data or calculation changes.
- No PDF engine replacement.
- No export-flow redesign.
- No invoice or quote changes.
- No schema changes.

## Acceptance Criteria
- Embedded TCPDF is loaded explicitly when present.
- `class_exists('TCPDF')` returns true after service-level engine loading when embedded TCPDF is present.
- Reporting PDF engine detector recognizes embedded TCPDF.
- Missing-engine fallback remains clear only when no embedded/vendor engine exists.
- Documentation reflects actual engine detection state.

## Validations Required
- `php scripts/php-lint.php --all`
- Runtime/manual checklist:
  - TCPDF embedded detected.
  - Download PDF Report enabled.
  - Reporting PDF downloads.
  - PDF opens correctly.
  - Missing-engine install message is not shown when embedded TCPDF exists.
