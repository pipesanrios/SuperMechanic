PHASE: 51D
NAME: Onboarding Base

==================================================
OBJETIVO
==================================================

Implementar una base de onboarding para que una instalación nueva
pueda configurarse de forma guiada y quedar operativa con negocio,
admin y estado inicial claros.

==================================================
SCOPE
==================================================
- onboarding state local
- detección de setup incompleto
- página/flujo base de onboarding
- pasos mínimos de configuración inicial
- estado visible de onboarding en admin

==================================================
OUT OF SCOPE
==================================================
- wizard multi-step avanzado
- tours interactivos
- importación masiva
- migraciones complejas
- CRM Pipeline avanzado
- asistentes IA

==================================================
ALLOWED FILES
==================================================
- includes/onboarding/class-onboarding-service.php (nuevo)
- includes/admin/class-onboarding-admin-controller.php (nuevo)
- includes/class-plugin.php
- includes/licensing/class-license-service.php
- includes/branding/class-branding-service.php
- assets/css/admin.css
- docs/contracts/51D.md
- docs/contracts/validation/51D-validation.md

==================================================
ESTADO DE ONBOARDING
==================================================

Debe poder detectar al menos:

- has_license
- has_branding_basic
- has_business
- has_business_admin
- is_onboarding_complete

Persistencia:
- WP options simple

==================================================
PASOS MÍNIMOS
==================================================

1. Verificar licencia
2. Verificar branding básico
3. Verificar negocio inicial
4. Verificar admin de negocio / membership
5. Mostrar siguiente acción recomendada

==================================================
SERVICE REQUERIDO
==================================================

Onboarding_Service

Métodos mínimos:
- get_onboarding_state()
- is_onboarding_complete()
- get_next_recommended_step()
- mark_onboarding_complete()
- reset_onboarding_state() // opcional si es limpio

==================================================
UI REQUERIDA
==================================================

Página admin:
- Super Mechanic → Onboarding

Mostrar:
- checklist de setup
- estado por paso
- siguiente acción recomendada
- CTA a páginas reales ya existentes:
  - License
  - Branding
  - Businesses
  - Roles & Access

Opcional:
- notice en dashboard/admin si onboarding incompleto

==================================================
CRITERIO DE IMPLEMENTACIÓN
==================================================
- no duplicar formularios ya existentes
- reutilizar páginas actuales
- onboarding solo orquesta/diagnostica
- no crear wizard complejo todavía
- UX clara y orientada a activar producto

==================================================
ACCEPTANCE CRITERIA
==================================================
- onboarding page visible
- checklist correcto
- next step correcto
- detecta instalación incompleta
- sin regresión funcional

==================================================
VALIDATIONS REQUIRED
==================================================
- php lint
- QA runner
- runtime manual
