PHASE: 53A
NAME: Operational Dashboards

==================================================
OBJETIVO
==================================================

Implementar dashboards operativos con métricas reales
del sistema para administración y operación diaria.

==================================================
SCOPE
==================================================
- dashboard admin principal
- métricas agregadas reales
- widgets básicos
- datos por negocio (business_id)
- refresh simple (no realtime complejo)

==================================================
OUT OF SCOPE
==================================================
- charts complejos (tipo BI)
- filtros avanzados
- multi-dashboard por rol
- analytics histórico profundo
- websockets / realtime

==================================================
ALLOWED FILES
==================================================
- includes/dashboard/class-dashboard-service.php (nuevo)
- includes/admin/class-dashboard-admin-controller.php (nuevo)
- includes/dashboard/class-dashboard-repository.php (nuevo)
- includes/class-plugin.php
- assets/css/admin.css
- docs/contracts/53A.md
- docs/contracts/validation/53A-validation.md

==================================================
MÉTRICAS REQUERIDAS
==================================================

Mínimo:

- total_clients
- total_vehicles
- active_processes
- completed_processes
- pending_processes
- active_webhooks
- notifications_today

Opcional (si fácil):
- failed_jobs
- retries_today

==================================================
SERVICE REQUERIDO
==================================================

Dashboard_Service

Métodos:
- get_dashboard_metrics($business_id = 0)
- get_recent_activity($limit = 10)

==================================================
REPOSITORY
==================================================

Dashboard_Repository

- consultas agregadas
- sin lógica de negocio
- eficiente (COUNT, GROUP BY)

==================================================
UI REQUERIDA
==================================================

Página:
Super Mechanic → Dashboard

Mostrar:
- tarjetas métricas (cards)
- actividad reciente (logs resumidos)
- layout limpio

==================================================
CRITERIO DE IMPLEMENTACIÓN
==================================================
- rápido de cargar
- consultas optimizadas
- datos reales (no mock)
- diseño simple pero claro
- reutilizar estilos admin existentes

==================================================
ACCEPTANCE CRITERIA
==================================================
- métricas correctas
- dashboard visible en admin
- sin regresión
- performance aceptable

==================================================
VALIDATIONS REQUIRED
==================================================
- php lint
- QA runner
- runtime manual
