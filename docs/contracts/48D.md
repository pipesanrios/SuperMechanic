PHASE: 48D
NAME: Persistencia ligera de preferencias

==================================================
OBJETIVO
==================================================

Permitir que cada usuario guarde preferencias ligeras de interfaz
para el dashboard, mejorando enfoque y usabilidad sin cambiar lógica operativa.

==================================================
SCOPE
==================================================

- persistencia por usuario
- bloques colapsados/expandidos
- visibilidad opcional de bloques secundarios
- preferencias ligeras de layout

==================================================
OUT OF SCOPE
==================================================
- personalización total del dashboard
- drag & drop complejo
- layouts por rol completamente separados
- cambios de lógica de negocio
- CRM Pipeline
- settings avanzados

==================================================
ALLOWED FILES
==================================================
- includes/dashboard/class-admin-dashboard-controller.php
- assets/js/admin-dashboard.js
- assets/css/admin.css
- docs/contracts/48D.md
- docs/contracts/validation/48D-validation.md

==================================================
CRITERIO DE IMPLEMENTACIÓN
==================================================
- persistencia simple por usuario
- usar user_meta o mecanismo WordPress equivalente
- no romper layout por defecto
- defaults seguros si no hay preferencias
- no introducir complejidad innecesaria

==================================================
OBJETIVO FUNCIONAL
==================================================

Preferencias iniciales sugeridas:

- collapsed_blocks: array de ids de bloques colapsados
- hidden_secondary_blocks: array de bloques secundarios ocultos
- compact_mode: boolean opcional (si aplica visualmente)

Solo para bloques secundarios/no críticos.
No permitir ocultar:
- centro de acción
- mi trabajo
- KPIs principales
- quick actions críticas

==================================================
ACCEPTANCE CRITERIA
==================================================
- usuario puede colapsar/ocultar ciertos bloques secundarios
- preferencia persiste al recargar
- defaults seguros funcionan
- bloques críticos siguen visibles
- sin regresión funcional

==================================================
VALIDATIONS REQUIRED
==================================================
- php lint
- QA runner
- runtime manual

==================================================
DOCS TO UPDATE
==================================================

- docs/tasks/2026-04-fase-48d-cierre.md

==================================================
TECHNICAL DEBT
==================================================
- sin drag & drop
- sin UI avanzada de preferencias
- sin sincronización entre dispositivos más allá de user_meta
