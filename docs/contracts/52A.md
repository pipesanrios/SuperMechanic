PHASE: 52A
NAME: Queue System Base

==================================================
OBJETIVO
==================================================

Implementar un sistema de colas básico para desacoplar
la ejecución de webhooks y notificaciones del flujo principal.

==================================================
SCOPE
==================================================
- tabla de cola
- repository de cola
- service de cola
- enqueue desde servicios existentes
- worker básico (procesamiento manual o cron simple)

==================================================
OUT OF SCOPE
==================================================
- workers distribuidos
- colas externas (Redis, RabbitMQ)
- retry avanzado (52B)
- dashboard de cola complejo
- priorización avanzada

==================================================
ALLOWED FILES
==================================================
- includes/queue/class-queue-installer.php (nuevo)

- docs/contracts/52A.md

==================================================
MODELO REQUERIDO
==================================================

Tabla: sm_queue

Campos mínimos:
- id
- job_type (notification, webhook)
- payload (JSON)
- status (pending, processing, completed, failed)
- attempts
- created_at
- processed_at
- last_error

==================================================
SERVICE REQUERIDO
==================================================

Queue_Service

Métodos:
- enqueue($job_type, $payload)
- process_next_job()
- process_batch($limit = 10)
- mark_completed($job_id)
- mark_failed($job_id, $error)

==================================================
INTEGRACIÓN REQUERIDA
==================================================

- Notification_Service:
  - reemplazar ejecución directa por enqueue

- Webhook_Service:
  - reemplazar dispatch directo por enqueue

IMPORTANTE:
- mantener fallback opcional si cola falla
