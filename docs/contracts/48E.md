PHASE: 48E
NAME: Roles & Access Management

==================================================
OBJETIVO
==================================================

Crear una pantalla administrativa para visualizar y gestionar
roles operativos y acceso interno de usuarios del sistema.

==================================================
SCOPE
==================================================

- listado de usuarios con rol operativo
- visualización de access summary
- administración básica de rol operativo
- validación de consistencia de acceso

==================================================
OUT OF SCOPE
==================================================
- editor complejo de capabilities
- matrix avanzada de permisos
- builder de roles
- cambios de lógica de negocio
- CRM Pipeline
- multi-tenant profundo

==================================================
ALLOWED FILES
==================================================
- includes/dashboard/class-admin-dashboard-controller.php si hace falta registrar submenu
- includes/users/class-admin-roles-controller.php (nuevo)
- includes/users/class-role-access-service.php (nuevo)
- assets/css/admin.css
- docs/contracts/48E.md
- docs/contracts/validation/48E-validation.md

==================================================
CRITERIO DE IMPLEMENTACIÓN
==================================================
- usar roles y capabilities actuales
- no duplicar lógica de acceso
- administración básica y segura
- lectura clara de inconsistencias
- no exponer capas internas a roles no autorizados

==================================================
OBJETIVO FUNCIONAL
==================================================

Pantalla `Roles & Access` debe mostrar por usuario:

- user_id
- display_name
- email
- WordPress role(s)
- operational role detected
- business_id (si existe)
- internal dashboard access: yes/no
- automation/logs access: yes/no
- status summary / warnings

Acciones básicas permitidas:
- asignar rol operativo:
  - sm_admin
  - sm_mechanic
- quitar rol operativo
- validar consistencia

==================================================
REGLAS
==================================================
- solo admin autorizado puede gestionar
- cambios por POST + nonce
- sanitización estricta
- no tocar capacidades base fuera de roles existentes
- no alterar usuarios cliente salvo remoción de acceso indebido si aplica

==================================================
ACCEPTANCE CRITERIA
==================================================
- pantalla de roles visible
- listado útil y claro
- cambios básicos de rol funcionan
- inconsistencias visibles
- sin regresión en dashboard/roles
