PHASE: 56A
NAME: Pre-SaaS Runtime Manual Audit

==================================================
OBJETIVO
==================================================

Ejecutar una auditoria runtime/manual integral del
estado actual de Super Mechanic antes de entrar a
una etapa SaaS hibrida, para clasificar modulos,
detectar gaps reales, confirmar flujos operativos
listos y documentar bloqueadores concretos.

==================================================
SCOPE
==================================================
- auditoria runtime/manual del plugin actual
- clasificacion por bloque: PASS / FAIL / PARTIAL / NOT TESTED
- verificacion de bloques operativos, portales, admin,
  integraciones, comercial/licencias y UX/calidad
- validacion explicita de:
  - Google Calendar
  - emails al cliente
  - inventario externo
  - PDF reporting
- uso de dataset demo existente cuando ayude
- evidencia cruzada desde:
  - runtime local WordPress
  - documentacion de cierres previos
  - wiring/codigo activo del runtime

==================================================
OUT OF SCOPE
==================================================
- nuevas features
- cambios de arquitectura
- correcciones funcionales
- cambios de schema
- refactor global
- trabajo sobre CRM Pipeline salvo hallazgo critico
- cierre de fase SaaS

- contratos/cierres previos para fases 50-55

==================================================
EXPECTED OUTPUTS
==================================================
- informe estructurado por bloques con clasificacion
  PASS / FAIL / PARTIAL / NOT TESTED
- lista concreta de gaps reales pre-SaaS
- lista concreta de modulos listos hoy
- lista concreta de bloqueadores reales
- recomendacion final de readiness pre-SaaS

==================================================
VALIDATION RULES
==================================================
- no implementar features nuevas
- no modificar arquitectura
- no expandir alcance fuera de la auditoria
- distinguir evidencia:
  - runtime/manual
  - tecnica/codigo
  - heredada/documentada
- no declarar PASS sin evidencia suficiente
- ejecutar `php scripts/php-lint.php --all`
- ejecutar QA Runner con el contrato de validacion 56A

==================================================
ACCEPTANCE CRITERIA
==================================================
- existe informe final completo por bloques
- cada bloque auditado tiene estado justificado
- los puntos sensibles A-D estan clasificados explicitamente
- los bloqueadores pre-SaaS son concretos y no aspiracionales
- el resultado distingue claramente listo hoy vs no listo
