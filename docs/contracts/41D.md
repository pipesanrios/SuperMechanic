PHASE: 41D
NAME: Asignación Operativa

==================================================
OBJETIVO
==================================================

Analizar carga operativa por usuario y generar
propuestas de redistribución sin ejecutar cambios.

==================================================
SCOPE
==================================================

- analizar carga por usuario
- detectar saturación y subutilización
- sugerir redistribución
- preparar estructura de asignación futura

==================================================
OUT OF SCOPE
==================================================

- cambios reales en asignaciones
- update de procesos/tareas
- cron
- nuevas tablas
- automatización ejecutable
- tocar CRM Pipeline

==================================================
ALLOWED FILES
==================================================

- includes/dashboard/class-workload-service.php
- includes/dashboard/class-admin-dashboard-controller.php
- docs/contracts/41D.md
- docs/contracts/validation/41D-validation.md

==================================================
OBJETIVO FUNCIONAL
==================================================

Detectar:

1. usuarios saturados (alto critical + workload)
2. usuarios disponibles (bajo workload)

Generar propuestas:

- mover carga de A → B
- balancear distribución
- sugerir asignación futura
