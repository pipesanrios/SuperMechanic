# TASK CONTRACT — 56P2-A Superadmin bootstrap

## Objective
Establish the initial superadmin bootstrap model so that the primary WordPress administrator becomes Mekvort superadmin on plugin activation/baseline initialization.

## Scope
- superadmin bootstrap assignment
- initial superadmin resolution
- safe persistence of superadmin baseline
- no UI-heavy changes

## In Scope
- detect/install initial superadmin from WordPress admin context
- persist superadmin baseline safely
- ensure other WP admins are not automatically promoted
- prepare foundation for later controlled superadmin assignment

## Out of Scope
- full superadmin UI management
- superadmin reassignment flows
- roles screen redesign
- Roles & Access UI changes
- CRM
- reset logic
- API
- language system changes
- schema changes unless strictly unavoidable

## Allowed Files
- super-mechanic.php if activation/bootstrap hook is involved
- includes/class-plugin.php if bootstrap wiring is needed
- includes/users/*
- docs/CURRENT_STATE.md
- docs/QA_REPORT.md

## Forbidden Files
- includes/crm/*
- includes/modules/*
- includes/class-admin-menu.php
- includes/class-settings.php
- REST/API files
- process/payment files
- Roles & Access UI files unless strictly required for non-visual wiring

## Constraints
- bootstrap only
- no automatic promotion of all WP admins
- preserve backward compatibility
- minimal scoped changes only

## Deliverables
- initial Mekvort superadmin baseline established
- only primary WP admin is auto-bootstrap superadmin
- docs updated for 56P2-A
- technical validation PASS

## Validation
- php scripts/php-lint.php --all
- runtime/manual check on:
  - fresh activation assigns only primary WP admin as superadmin
  - other WP admins are not auto-promoted
  - no admin regression
