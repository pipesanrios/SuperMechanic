PHASE: 48A
NAME: Configuración operativa por negocio

==================================================
OBJETIVO
==================================================

Permitir configurar thresholds y activación de señales operativas
por negocio (business_id), manteniendo lógica existente.

==================================================
SCOPE
==================================================

- configuración por business_id
- thresholds operativos configurables
- activación/desactivación de señales
- acceso centralizado a configuración

==================================================
OUT OF SCOPE
==================================================
- UI compleja de settings
- panel avanzado de configuración
- multi-tenant profundo
- cambios de lógica de negocio
- CRM Pipeline
- automatización avanzada

==================================================
ALLOWED FILES
==================================================
- includes/config/class-operational-config-service.php (nuevo)
- includes/dashboard/class-workload-service.php
- includes/automation/class-operational-rules-service.php (solo lectura config)
- docs/contracts/48A.md
- docs/contracts/validation/48A-validation.md

==================================================
CRITERIO DE IMPLEMENTACIÓN
==================================================
- configuración simple y clara
- defaults seguros
- sin romper comportamiento actual
- acceso centralizado (no duplicación)
- fallback automático si no existe config

==================================================
CONFIGURACIONES INICIALES
==================================================

thresholds:

- overdue_tasks_threshold (default: 1)
- delayed_process_threshold (default: 1)
- critical_signal_threshold (default: 2)
- user_saturation_threshold (default: 3)

flags:

- enable_recommendations (true)
- enable_internal_flags (true)
- enable_escalation (true)

==================================================
ACCESO A CONFIG
==================================================

Debe existir un service:

Operational_Config_Service

Métodos:

- get($business_id, $key, $default = null)
- get_all($business_id)
- get_threshold($business_id, $key)
- is_enabled($business_id, $flag)

==================================================
ACCEPTANCE CRITERIA
==================================================

- sistema usa defaults si no hay config
- thresholds pueden cambiar comportamiento (sin romper lógica)
- flags pueden apagar features visuales (no lógica base)
- sin regresión funcional
- acceso centralizado correcto
