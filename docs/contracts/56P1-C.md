# TASK CONTRACT — 56P1-C I18N helper base

## Objective
Implement a centralized i18n helper foundation for Mekvort, with safe defaults, persisted current language, available language registry, and a minimal translation access layer.

## Scope
- central i18n helper
- current language getter
- current language setter
- available languages registry
- persisted default/current language resolution
- minimal translate helper baseline

## In Scope
- helper/service for language resolution
- wp_options-backed persistence
- bundled languages support:
  - en_US
  - es_ES
  - it_IT
- default fallback behavior
- safe integration point for future language expansion

## Out of Scope
- full translation coverage
- mass string replacement
- frontend language switching
- menu rename
- roles logic
- CRM
- reset logic
- API localization
- database schema changes

## Allowed Files
- includes/helpers/class-i18n-helper.php
- includes/class-settings.php only if strictly needed to connect existing setting to helper
- docs/CURRENT_STATE.md
- docs/QA_REPORT.md

## Forbidden Files
- includes/class-admin-menu.php
- includes/users/*
- includes/crm/*
- includes/modules/*
- database/schema files
- REST/API files
- Roles & Access files
- process/payment/CRM files

## Constraints
- helper must be centralized
- minimal scoped changes only
- no full translation rollout
- preserve backward compatibility
- English remains the default baseline language

## Deliverables
- central i18n helper created or stabilized
- helper methods available for future phases
- current language persisted/resolved safely
- docs updated for 56P1-C
- technical validation PASS

## Validation
- php scripts/php-lint.php --all
- helper exists and loads
- runtime/manual check on:
  - helper uses persisted setting
  - fallback to English works
  - available languages list is correct
- no admin regression
