PHASE: 41A
NAME: Reglas Automáticas Internas

==================================================
OBJETIVO
==================================================

Introducir una capa de automatización operativa interna basada en reglas seguras,
sin crear automatizaciones externas ni acciones irreversibles.

==================================================
SCOPE
==================================================

- motor interno de reglas operativas
- evaluación de condiciones sobre datos existentes
- generación de sugerencias o flags automáticos
- sin modificar procesos core fuera de reglas explícitas

==================================================
OUT OF SCOPE
==================================================

- emails
- WhatsApp
- notificaciones push
- nuevas tablas
- cron nuevo
- automatización externa
- cambios destructivos automáticos
- auto-cierre de procesos
- auto-edición de entidades críticas

- docs/contracts/41A.md
- docs/contracts/validation/41A-validation.md

==================================================
FORBIDDEN FILES
==================================================

- includes/modules/*
- schema/database
- procesos core fuera de lectura
- appointments core
- invoices/payments core

==================================================
OBJETIVO FUNCIONAL
==================================================

Crear reglas automáticas internas como:

1. si una tarea está overdue y sigue abierta:
   - marcarla como escalada operativamente

2. si un proceso lleva demasiado tiempo activo:
   - señal de retraso reforzada

3. si un usuario tiene carga crítica elevada:
   - generar flag de saturación

4. si existen múltiples señales críticas:
   - mostrar estado operativo elevado en dashboard

==================================================
CRITERIO DE IMPLEMENTACIÓN
==================================================

- reutilizar señales existentes
- no duplicar lógica
- no recalcular persistidas si ya existen
- usar scheduler actual solo si encaja en scope
- no crear nueva infraestructura innecesaria

==================================================
ACCEPTANCE CRITERIA
==================================================

- existe capa de reglas automáticas internas
- las reglas usan datos ya existentes
- no hay nuevas tablas
- no hay automatización externa
- dashboard refleja señales automáticas nuevas si aplican
- sistema sigue estable y sin regresión

- validación manual runtime
- sin regresión dashboard / CRM Pipeline / workload
