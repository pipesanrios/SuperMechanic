PHASE: 55B
NAME: Public API Formalization

==================================================
OBJETIVO
==================================================

Implementar una API REST pública, estable y versionada
para acceso externo seguro a recursos del sistema.

==================================================
SCOPE
==================================================
- namespace API formal
- endpoints REST versionados
- ownership validation
- auth compatible con WordPress
- estructura consistente de respuesta

==================================================
OUT OF SCOPE
==================================================
- OAuth complejo
- rate limiting avanzado
- GraphQL
- webhooks de salida (otra fase)
- API pública completa de todos los módulos
- SDKs externos

==================================================
ALLOWED FILES
==================================================
- includes/api/class-api-loader.php (nuevo)
- includes/api/controllers/ (nuevo)
- includes/api/middleware/ (nuevo si hace falta)
- includes/class-plugin.php
- docs/contracts/55B.md
- docs/contracts/validation/55B-validation.md

==================================================
NAMESPACE REQUERIDO
==================================================

/wp-json/sm/v1/

==================================================
ENDPOINTS MÍNIMOS
==================================================

GET /clients
GET /vehicles
GET /processes
GET /processes/{id}
GET /invoices
GET /reporting/summary

Opcional si queda limpio:
POST /quotes/{id}/approve

==================================================
SEGURIDAD
==================================================
- usar auth compatible con WordPress:
  - sesión autenticada admin
  - Application Passwords si aplica
- ownership validation obligatoria
- no exponer datos fuera del tenant/business
- sanitización estricta de parámetros

==================================================
RESPUESTAS
==================================================

Formato consistente tipo:
{
  "success": true,
  "data": [...],
  "meta": {...}
}

Errores:
{
  "success": false,
  "error": "message"
}

==================================================
CRITERIO DE IMPLEMENTACIÓN
==================================================
- controllers delgados
- service/repository existentes como fuente
- no SQL en controllers
- endpoints rápidos y claros
- estructura extensible

==================================================
ACCEPTANCE CRITERIA
==================================================
- namespace activo
- endpoints responden
- ownership correcto
- reporting summary accesible
- sin regresión admin/frontend

==================================================
VALIDATIONS REQUIRED
==================================================
- php lint
- QA runner
- runtime manual con llamadas REST
