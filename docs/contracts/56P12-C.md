# TASK CONTRACT - 56P12-C Vehicle creation from catalog

## Objective
Allow customer vehicles to be created or edited faster from reusable vehicle catalog records.

## Scope
- admin vehicle create/edit UI integration
- optional catalog selector
- active catalog record selection scoped by business
- auto-fill compatible vehicle fields from selected catalog record
- preserve manual override after selection
- persist catalog reference only if the existing vehicle schema already supports it or a strictly required scoped schema update is explicitly added

## In Scope
- vehicle admin create/edit page integration
- catalog options loaded through `Vehicle_Catalog_Service`
- JavaScript auto-fill behavior for matching vehicle fields
- server-side save support for catalog-selected compatible fields
- nonce/capability behavior inherited from vehicle admin save flow
- documentation updates

## Out of Scope
- CSV import
- external inventory connector
- CRM/reset/API changes
- process/payment logic
- frontend portal changes
- forced locking of catalog-derived fields
- broad schema redesign

## Allowed Files
- `includes/vehicles/*`
- `includes/admin/*`
- `assets/js/*`
- `assets/css/*` only if needed
- `docs/CURRENT_STATE.md`
- `docs/QA_REPORT.md`
- `docs/tasks/*`

## Forbidden Files
- `includes/crm/*`
- `includes/users/*`
- `includes/modules/*`
- REST/API files
- external inventory connector
- CSV import
- process/payment logic
- frontend portal
- schema changes unless strictly required and explicitly scoped before implementation

## Validation Contract
- `docs/contracts/validation/56P12-C-validation.md`

## Constraints
- Controller -> Service -> Repository -> Database
- no SQL in controllers
- do not duplicate catalog logic
- use `Vehicle_Catalog_Service`
- preserve business scope
- preserve manual override
- no CSV import
- no external connector
- no CRM/reset/API changes

## Acceptance Criteria
- Vehicle create/edit admin UI exposes an optional catalog selector.
- Selector lists active catalog records scoped by the active/requested business.
- Selecting a catalog record can auto-fill compatible vehicle fields.
- VIN, plate/license plate, color, mileage and client/owner remain manual.
- User can manually override catalog-filled fields.
- Catalog reference is persisted only if safe under existing or explicitly scoped schema support.
- Existing vehicle create/edit behavior remains backward compatible.
