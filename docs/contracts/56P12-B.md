# TASK CONTRACT - 56P12-B Vehicle catalog admin UI

## Objective
Create an admin UI for managing business-scoped vehicle catalog records.

## Scope
- admin UI only
- list/create/edit/deactivate vehicle catalog records
- reuse `Vehicle_Catalog_Service`
- preserve business scope

## In Scope
- Vehicle Catalog submenu/page under Super Mechanic
- table-style admin list
- create/edit form
- deactivate action
- nonce and capability checks
- input sanitization and output escaping
- documentation updates

## Out of Scope
- CSV import
- creating customer vehicles from catalog records
- external inventory connector
- VIN decoding
- REST/API changes
- schema changes unless strictly required
- CRM, users/roles, process/payment logic
- frontend portal

## Allowed Files
- `includes/admin/*`
- `includes/vehicles/*`
- `includes/class-admin-menu.php` only if needed to register menu/submenu
- `assets/css/*`
- `docs/CURRENT_STATE.md`
- `docs/QA_REPORT.md`
- `docs/tasks/*`

## Forbidden Files
- `includes/crm/*`
- `includes/users/*`
- `includes/modules/*`
- REST/API files
- process/payment logic
- frontend portal
- schema changes unless strictly required

## Constraints
- Controller -> Service -> Repository -> Database
- admin controller must use `Vehicle_Catalog_Service` only for catalog persistence
- no SQL in admin controller
- no duplicated catalog persistence logic
- tenant/business scope must be preserved
- no CSV import
- no vehicle creation from catalog

## Acceptance Criteria
- Vehicle Catalog admin page is available under Super Mechanic.
- Admin can list catalog records with required columns.
- Admin can create catalog records.
- Admin can edit catalog records.
- Admin can deactivate catalog records.
- All writes use nonce and capability checks.
- Output is escaped and input is sanitized.
- Catalog persistence goes through `Vehicle_Catalog_Service`.

- manual/runtime:

  - records remain business-scoped
