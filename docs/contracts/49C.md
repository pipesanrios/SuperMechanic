PHASE: 49C
NAME: Roles & Access UI by Business

==================================================
OBJETIVO
==================================================

Extender la pantalla de Roles & Access para permitir
gestionar membresías por negocio (crear, editar, activar,
desactivar y definir primaria), sin depender solo de roles WP.

==================================================
SCOPE
==================================================

- UI para visualizar memberships por usuario
- asignación de business_id + rol operativo
- activar/desactivar membership
- definir membership primaria
- integración con Role_Access_Service

==================================================
OUT OF SCOPE
==================================================
- transferencias complejas entre negocios (49D)
- permisos avanzados por acción
- editor de capabilities
- cambios en lógica de negocio
- CRM Pipeline

==================================================
ALLOWED FILES
==================================================
- includes/users/class-business-membership-service.php
- includes/users/class-business-membership-repository.php (si requiere write methods)
- includes/users/class-role-access-service.php
- includes/admin/class-roles-access-controller.php (nuevo o existente)
- assets/js/admin-roles-access.js (nuevo/modificado)
- assets/css/admin.css
- docs/contracts/49C.md
- docs/contracts/validation/49C-validation.md

==================================================
FUNCIONALIDAD REQUERIDA
==================================================

Para cada usuario:

1. Mostrar memberships existentes:
   - business_id
   - operational_role
   - status (active/inactive)
   - is_primary

2. Permitir:
   - agregar membership
   - cambiar rol operativo
   - cambiar business_id
   - activar/desactivar
   - marcar como primaria

3. Reglas:

- solo 1 membership primaria por usuario
- membership primaria debe ser active
- si se marca nueva primaria → desmarcar anterior
- no permitir duplicados exactos (user + business)

==================================================
ACCIONES UI
==================================================

- "Add membership"
- "Set as primary"
- "Deactivate"
- "Activate"
- "Change role"
- "Remove membership" (opcional soft delete o inactive)

==================================================
BACKEND (SERVICE)
==================================================

Agregar métodos:

- create_membership($user_id, $business_id, $role)
- update_membership_role($membership_id, $role)
- set_membership_status($membership_id, $status)
- set_primary_membership($membership_id)
- remove_membership($membership_id)

==================================================
SEGURIDAD
==================================================

- capability requerida: sm_manage_plugin
- nonce en todas las acciones
- sanitización estricta
- validar business_id existente

==================================================
CRITERIO DE IMPLEMENTACIÓN
==================================================

- NO romper UI actual → extenderla
- NO romper roles WP actuales
- NO tocar lógica de negocio
- mantener WP_List_Table o estructura actual

==================================================
ACCEPTANCE CRITERIA
==================================================

- se pueden crear memberships
- se pueden editar roles por negocio
- primaria se gestiona correctamente
- no hay duplicados
- warnings actuales desaparecen al asignar memberships
- sin regresión dashboard
