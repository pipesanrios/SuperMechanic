PHASE: 55E1
NAME: Commercial Hooks

==================================================
OBJETIVO
==================================================

Crear una capa centralizada de hooks comerciales para extender el sistema
sin modificar el core de negocio existente.

==================================================
SCOPE
==================================================
- servicio central de hooks comerciales
- naming estandar de hooks
- payload comercial consistente
- integracion en puntos comerciales existentes

==================================================
OUT OF SCOPE
==================================================
- nuevos flujos de negocio
- cambios de schema
- cambios de CRM Pipeline
- reemplazo de Event_Dispatcher existente

==================================================
ALLOWED FILES
==================================================
- includes/commercial/class-commercial-hooks-service.php (nuevo)
- includes/quotes/class-quote-service.php
- includes/invoices/class-invoice-service.php
- includes/processes/class-process-service.php
- docs/contracts/55E1.md
- docs/contracts/validation/55E1-validation.md

==================================================
IMPLEMENTACION REQUERIDA
==================================================

1. Crear `Commercial_Hooks_Service`:
- `dispatch($hook_name, $payload)`
- `get_supported_hooks()`

2. Hooks soportados:
- `sm_quote_created`
- `sm_quote_approved`
- `sm_invoice_created`
- `sm_invoice_paid`
- `sm_payment_created`
- `sm_process_completed`

3. Integracion requerida:
- Quote service
- Invoice service (incluye capa de pagos vigente)
- Process service

4. Payload estandar:
- `business_id`
- `entity_id`
- `entity_type`
- `data`

==================================================
ACCEPTANCE CRITERIA
==================================================
- capa central de hooks creada y reusable
- hooks requeridos disparados desde servicios correspondientes
- payload comercial consistente aplicado
- sin regresion funcional
