PHASE: 43A
NAME: Ejecución manual guiada por reglas

==================================================
OBJETIVO
==================================================

Permitir ejecutar manualmente acciones recomendadas por reglas activadas,
desde el dashboard, usando la infraestructura ya creada en 42A–42E.

==================================================
SCOPE
==================================================

- conectar reglas activadas con acciones manuales ejecutables
- exponer botones seguros desde dashboard
- reutilizar ejecución ya existente de 42A/42B/42C
- no introducir automatización autónoma

==================================================
OUT OF SCOPE
==================================================

- cron
- autoejecución
- triggers automáticos
- nuevas tablas
- persistencia compleja de reglas
- edición avanzada de reglas
- rollback automático
- tocar CRM Pipeline core

==================================================
ALLOWED FILES
==================================================

- includes/automation/class-operational-rules-service.php
- includes/dashboard/class-workload-service.php
- includes/dashboard/class-admin-dashboard-controller.php
- assets/css/admin.css
- docs/contracts/43A.md
- docs/contracts/validation/43A-validation.md

==================================================
FORBIDDEN FILES
==================================================

- includes/modules/*
- schema/database
- includes/crm/* salvo reutilización indirecta ya existente
- includes/processes/*
- includes/appointments/*
- invoices/payments core

==================================================
OBJETIVO FUNCIONAL
==================================================

Cuando una regla esté `triggered = true`, el sistema debe poder mostrar:

- acción sugerida
- acción ejecutable manualmente
- contexto de riesgo / impacto
- ruta segura de ejecución

Reglas iniciales soportadas:

1. overdue_tasks_cleanup
   - acción manual guiada:
     - `bulk_resolve`

2. critical_saturation_rebalance
   - acción manual guiada:
     - `bulk_reassign`

3. multi_critical_alert
   - acción manual guiada:
     - abrir centro de acción / intervención operativa
     - no ejecutar mutación si no existe handler seguro

==================================================
CRITERIO DE IMPLEMENTACIÓN
==================================================

- reutilizar handlers ya existentes
- no duplicar lógica de ejecución
- no recalcular reglas fuera del servicio
- no ejecutar nada automáticamente

==================================================
ESTRUCTURA DE SALIDA
==================================================

```php
[
  'guided_actions' => [

      'executable' => true|false,

    ]
  ],
  'summary' => [
    'total' => int,
