# TASK CONTRACT — 56P8-A Email Trigger System

## Objective
Implement a centralized email trigger system that listens to key domain events and dispatches notification intents without coupling to templates or delivery providers.

## Scope
- event → trigger → notification dispatch
- reusable email trigger service
- integration points in existing flows

## In Scope
- Email_Trigger_Service (core dispatcher)
- hook points in:
  - process status change
  - quote status change (approved/rejected)
  - invoice status change (paid/partial)
- basic payload structure for notifications

## Out of Scope
- email templates (56P8-B)
- SMTP / provider integration (56P8-C)
- UI configuration for notifications
- queue system (future)

## Allowed Files
- includes/services/*
- includes/modules/* (only where event hooks are needed)
- includes/helpers/*
- docs/CURRENT_STATE.md
- docs/QA_REPORT.md

## Forbidden Files
- frontend (css/js)
- dashboard controllers
- CRM redesign
- database schema changes (unless strictly minimal)

## Constraints
- no business logic duplication
- event-driven approach
- reusable and extendable
- no hardcoded HTML emails

## Deliverables
- centralized trigger service
- hooks connected to real events
- structured notification payloads

## Validation
- php scripts/php-lint.php --all
- manual trigger simulation
