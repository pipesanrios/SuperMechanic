PHASE: 49D
NAME: Membership Transfers Between Businesses

==================================================
OBJETIVO
==================================================

Permitir transferir usuarios entre negocios mediante
gestión de memberships, manteniendo consistencia y
sin pérdida de información operativa.

==================================================
SCOPE
==================================================

- transferencia de membership a otro negocio
- cambio de primaria automático
- mantenimiento de histórico (no borrar registros)
- validación de integridad

==================================================
OUT OF SCOPE
==================================================
- migración de datos operativos (procesos, tareas, etc.)
- reasignación automática de workload
- UI avanzada multi-step
- CRM Pipeline

==================================================
ALLOWED FILES
==================================================
- includes/users/class-business-membership-service.php
- includes/users/class-business-membership-repository.php
- includes/admin/class-roles-access-controller.php
- assets/js/admin-roles-access.js
- docs/contracts/49D.md
- docs/contracts/validation/49D-validation.md

==================================================
TIPOS DE TRANSFERENCIA
==================================================

1. Transferencia simple:
   - usuario pasa a otro business
   - nueva membership activa
   - anterior se desactiva

2. Transferencia con mantenimiento:
   - usuario queda en ambos negocios
   - se cambia primaria

==================================================
REGLAS
==================================================

- nunca eliminar histórico
- no duplicar memberships activas iguales
- siempre debe haber máximo 1 primaria
- si se transfiere primaria → nueva primaria obligatoria
- validar business_id destino

==================================================
SERVICE REQUERIDO
==================================================

Agregar métodos:

- transfer_user_to_business($user_id, $target_business_id, $role, $mode)

Donde:

$mode:
- replace (mover)
- add (agregar sin eliminar anterior)

==================================================
COMPORTAMIENTO
==================================================

mode = replace:
- desactivar memberships actuales activas
- crear o reactivar membership en destino
- marcar como primaria

mode = add:
- crear membership adicional
- no tocar otras
- opcional marcar primaria

==================================================
UI
==================================================

Agregar acción:

- "Transfer / Move user"

Opciones:
- seleccionar negocio destino
- seleccionar rol
- seleccionar modo:
  - replace
  - add

ACCEPTANCE CRITERIA

- no hay duplicados activos
- warnings desaparecen tras transferencia válida
