PHASE: 49B
NAME: Super Admin / Global Access

==================================================
OBJETIVO
==================================================

Formalizar la capa de acceso global para distinguir
usuarios con control total del sistema de usuarios
limitados a uno o más negocios por membresía operativa.

==================================================
SCOPE
==================================================

- detección centralizada de super admin global
- access summary global vs business-scoped
- integración con memberships por negocio
- lectura segura de alcance multi-business

==================================================
OUT OF SCOPE
==================================================
- UI completa de administración de roles
- transferencias
- editor complejo de permisos
- cambios de lógica de negocio
- CRM Pipeline
- migraciones masivas

==================================================
ALLOWED FILES
==================================================
- includes/users/class-business-membership-service.php
- includes/users/class-role-access-service.php (nuevo)
- includes/dashboard/class-admin-dashboard-controller.php solo si hace falta lectura/resumen
- docs/contracts/49B.md
- docs/contracts/validation/49B-validation.md

==================================================
CRITERIO DE IMPLEMENTACIÓN
==================================================
- no romper roles actuales
- no depender solo de business_id legacy
- usar capa centralizada de access summary
- super admin global debe resolver acceso a todos los negocios
- fallback seguro para usuarios no globales

==================================================
REGLAS DE ACCESO
==================================================

1. Super admin global
- usuario con rol/capability global autorizado
- acceso total a todos los negocios
- puede visualizar alcance multi-business

2. Usuario con membresía por negocio
- acceso restringido a sus memberships activas
- si tiene una primaria, esa puede ser el default operativo

3. Usuario sin membresía activa y sin acceso global
- no debe considerarse operador válido multi-business

==================================================
SERVICE REQUERIDO
==================================================

Crear:
- Role_Access_Service

Métodos mínimos:

- is_global_super_admin($user_id)
- get_access_scope($user_id)
- get_accessible_business_ids($user_id)
- can_access_business($user_id, $business_id)
- get_default_business_id($user_id)

==================================================
SALIDA ESPERADA DE ACCESS SCOPE
==================================================

Ejemplo:

[
  'user_id' => 1,
  'is_global' => true,
  'accessible_business_ids' => [1,2,3],
  'default_business_id' => 1,
  'memberships' => [...],
  'source' => 'global_role|membership'
]

==================================================
ACCEPTANCE CRITERIA
==================================================

- super admin global se resuelve correctamente
- usuarios normales solo acceden a negocios permitidos
- default business se resuelve de forma segura
- integración con memberships funciona
- sin regresión funcional
