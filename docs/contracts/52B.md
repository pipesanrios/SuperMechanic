PHASE: 52B
NAME: Retry System Base

==================================================
OBJETIVO
==================================================

Implementar un sistema de reintentos para jobs fallidos de cola,
con backoff básico y control de intentos máximos.

==================================================
SCOPE
==================================================
- retry count
- max attempts
- next_retry_at
- backoff básico
- reintento automático desde worker

==================================================
OUT OF SCOPE
==================================================
- backoff exponencial complejo
- prioridades avanzadas
- dead-letter queue separada
- UI avanzada de monitoreo
- alertas automáticas

==================================================
ALLOWED FILES
==================================================
- includes/queue/class-queue-installer.php

==================================================
MODELO REQUERIDO
==================================================

Extender sm_queue con campos mínimos:
- max_attempts
- next_retry_at

Mantener:
- attempts
- status
- last_error

==================================================
REGLAS DE RETRY
==================================================

- pending → processing → completed
- si falla:
  - incrementar attempts
  - si attempts < max_attempts:
    - volver a pending
    - programar next_retry_at
  - si attempts >= max_attempts:
    - status = failed

==================================================
BACKOFF BÁSICO
==================================================

Ejemplo simple:
- intento 1 fallido → retry en 5 min
- intento 2 fallido → retry en 15 min
- intento 3 fallido → retry en 30 min

Puede ser lineal o escalonado simple, pero claro en código.

==================================================
SERVICE REQUERIDO
==================================================

Queue_Service debe soportar:
- schedule_retry($job_id, $attempts, $error)
- get_retry_delay_seconds($attempts)
- process_next_job() respetando next_retry_at

==================================================
CRITERIO DE IMPLEMENTACIÓN
==================================================
- no romper 52A
- retries solo para jobs elegibles
- no procesar jobs antes de next_retry_at
- mantener logs de error simples

==================================================
ACCEPTANCE CRITERIA
==================================================
- jobs fallidos reintentan correctamente
- jobs agotados quedan en failed
- next_retry_at se respeta
- sin regresión en queue base
