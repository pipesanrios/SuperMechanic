PHASE: 50B
NAME: Operational Notification Triggers

==================================================
OBJETIVO
==================================================

Conectar el sistema base de notificaciones con eventos
operativos reales del sistema, usando triggers claros,
controlados y desacoplados.

==================================================
SCOPE
==================================================

- triggers operativos iniciales
- integración con Notification_Service
- mapeo evento -> tipo de notificación
- disparo controlado desde services existentes

==================================================
OUT OF SCOPE
==================================================
- UI de notificaciones
- bandeja de notificaciones
- configuración avanzada por usuario
- WhatsApp / SMS
- CRM Pipeline complejo nuevo
- automatización avanzada

==================================================
ALLOWED FILES
==================================================
- includes/notifications/class-notification-service.php
- includes/notifications/class-notification-template-service.php
- includes/notifications/class-email-delivery-service.php
- includes/users/class-business-membership-service.php

- includes/processes/* solo si ya existe service claro y hace falta integrar trigger
- docs/contracts/50B.md
- docs/contracts/validation/50B-validation.md

==================================================
CRITERIO DE IMPLEMENTACIÓN
==================================================
- triggers desde services, no desde controllers
- no duplicar disparos
- no bloquear flujo principal si falla email
- mapping claro de eventos a notification types

==================================================
EVENTOS INICIALES
==================================================

Implementar al menos triggers para:

- membership_created
- membership_updated
- user_transferred
- overdue_alert_detected
- critical_signal_detected

Si el código real ya permite eventos de procesos/citas de forma limpia, agregar:
- appointment_created
- process_status_critical

==================================================
ACCEPTANCE CRITERIA
==================================================
- eventos disparan notificaciones correctas
- no hay duplicación evidente
- flujo principal sigue estable
- sistema sigue desacoplado
