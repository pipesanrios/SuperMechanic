PHASE: 50A
NAME: Notification System (Email Base)

==================================================
OBJETIVO
==================================================

Implementar un sistema base de notificaciones por email
desacoplado de la lógica de negocio, reutilizable y extensible.

==================================================
SCOPE
==================================================

- Notification Service central
- envío de emails vía wp_mail
- sistema básico de templates
- disparo manual/controlado de notificaciones
- integración inicial con users + memberships

==================================================
OUT OF SCOPE
==================================================

- WhatsApp / SMS
- Webhooks
- Automatización avanzada
- Configuración UI compleja
- Editor visual de templates

==================================================
ALLOWED FILES
==================================================

- includes/notifications/class-notification-service.php
- includes/notifications/class-notification-template-service.php
- includes/notifications/class-email-delivery-service.php
- includes/users/class-business-membership-service.php (solo integración ligera)
- docs/contracts/50A.md
- docs/contracts/validation/50A-validation.md

==================================================
ARQUITECTURA REQUERIDA
==================================================

Separar claramente:

1. Notification_Service
   - orquesta
   - decide qué enviar y a quién

2. Notification_Template_Service
   - obtiene subject/body
   - variables dinámicas

3. Email_Delivery_Service
   - envía via wp_mail

==================================================
FUNCIONALIDAD MÍNIMA
==================================================

Implementar:

Notification_Service:
- send_notification($type, $user_id, $data = [])

Notification_Template_Service:
- get_template($type)
- render_template($template, $data)

Email_Delivery_Service:
- send_email($to, $subject, $body)

==================================================
TIPOS DE NOTIFICACIÓN INICIALES
==================================================

- user_assigned_to_business
- user_transferred
- membership_created
- membership_updated

==================================================
DATOS DISPONIBLES
==================================================

- user (email, name)
- business (id, name si existe)
- role
- status

==================================================
REGLAS
==================================================

- NO hardcodear lógica en controllers
- usar services
- permitir agregar nuevos tipos fácilmente
- sanitizar output
- evitar dependencias externas

==================================================
ACCEPTANCE CRITERIA
==================================================

- se puede enviar email real con wp_mail
- templates renderizan variables
- sistema desacoplado funciona
- integración básica con memberships operativa

==================================================
VALIDATIONS REQUIRED
==================================================

- php lint
- QA runner
- prueba manual envío de email
