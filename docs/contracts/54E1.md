PHASE: 54E.1
NAME: Embedded PDF Engine

==================================================
OBJETIVO
==================================================

Integrar un motor PDF embebido dentro de Super Mechanic para habilitar
la descarga de reportes PDF sin depender de plugins externos de WordPress.

==================================================
SCOPE
==================================================
- resolver engine PDF embebido para Reporting
- mantener fallback claro si el engine no carga
- mantener endpoint/admin reporting estable

==================================================
ALLOWED FILES
==================================================
- includes/reporting/class-report-pdf-service.php
- includes/admin/class-reporting-admin-controller.php
- composer.json (si aplica)
- vendor/* (si aplica)
- includes/libs/pdf/* (si aplica)
- super-mechanic.php (solo si bootstrap de engine lo requiere)
- docs/contracts/54E1.md

==================================================
OUT OF SCOPE
==================================================
- cambios CRM Pipeline
- charts complejos
- rediseño de reporting
- cambios de schema
- dependencias en plugins externos

==================================================
ACCEPTANCE CRITERIA
==================================================
- el boton de Reporting sigue visible y estable
- el PDF puede descargarse sin plugin externo si engine embebido carga
- fallback admin claro si el engine falla
- no regresion en reporting/admin

==================================================
VALIDATIONS REQUIRED
==================================================
- php scripts/php-lint.php --all
- php scripts/qa-runner.php --contract=docs/contracts/validation/54E-validation.md --output=text
- runtime manual (descarga + lectura visual PDF)

==================================================
TECHNICAL DEBT
==================================================
- mantener fallback mientras no exista validacion runtime automatizada
