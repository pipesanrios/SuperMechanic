PHASE: 46A
NAME: Query optimization

==================================================
OBJETIVO
==================================================

Optimizar consultas y agregaciones pesadas del dashboard y capas operativas
para reducir tiempo de carga sin alterar lógica funcional.

==================================================
SCOPE
==================================================

- identificar queries repetidas o costosas
- consolidar lecturas redundantes
- reducir N+1
- optimizar acceso a datos en dashboard/automation/logs

==================================================
OUT OF SCOPE
==================================================

- transients persistentes
- lazy loading UI
- cambios de UX grandes
- indexación DB
- profiling avanzado
- nuevas tablas

==================================================
ALLOWED FILES
==================================================

- includes/dashboard/class-workload-service.php
- includes/dashboard/class-admin-dashboard-controller.php
- includes/automation/class-operational-rules-service.php
- includes/automation/class-execution-log-repository.php
- includes/automation/class-execution-log-service.php
- includes/crm/class-crm-task-repository.php
- docs/contracts/46A.md
- docs/contracts/validation/46A-validation.md

==================================================
OBJETIVO FUNCIONAL
==================================================

- reducir recomputación y queries repetidas
- mantener mismos resultados
- mantener dashboard y logs consistentes
- mejorar tiempo de carga perceptible

==================================================
CRITERIO DE IMPLEMENTACIÓN
==================================================

- no cambiar lógica de negocio
- no cambiar resultados funcionales
- preferir consolidación de lecturas
- preferir payloads compartidos
- evitar consultas duplicadas por request

==================================================
ACCEPTANCE CRITERIA
==================================================

- dashboard carga perceptiblemente más rápido
- logs siguen funcionando
- sin regresión 43A–45E
- mismos resultados funcionales
- menos carga redundante por request

VALIDATIONS REQUIRED

- QA runner
- runtime manual
- comparación antes/después
