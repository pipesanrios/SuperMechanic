PHASE: 40B
NAME: Workload Operativo por Usuario

==================================================
OBJETIVO
==================================================

Construir una vista operativa unificada por usuario que muestre:
- tareas
- alertas
- procesos
- citas

con priorización clara y ordenada.

==================================================
SCOPE
==================================================

- agregación multi-módulo por `assigned_user_id`
- clasificación en:
  - CRITICAL
  - WARNING
  - NORMAL
- orden por prioridad y fecha
- exposición en dashboard "Mi trabajo"

==================================================
OUT OF SCOPE
==================================================

- creación de nuevas tablas
- cambios en CRM core
- cambios en scheduler
- cambios en alert system
- automatizaciones nuevas
- cron
- emails
- notificaciones

==================================================
ALLOWED FILES
==================================================

- includes/dashboard/class-workload-service.php
- includes/dashboard/class-admin-dashboard-controller.php
- assets/css/admin.css (opcional)

==================================================
FORBIDDEN FILES
==================================================

- includes/crm/*
- includes/processes/*
- includes/appointments/*
- scheduler
- alerts core

==================================================
DEPENDENCIES
==================================================

- Crm_Task_Service / Repository
- Crm_Alert_Repository
- Process repository
- Appointments source

==================================================
INPUTS
==================================================

- assigned_user_id
- filtros opcionales (future-safe)

==================================================
EXPECTED OUTPUT
==================================================

estructura:

- critical[]
- warning[]
- normal[]

cada item:

- type (task, process, appointment)
- title
- url
- date
- priority
- source_id

==================================================
ACCEPTANCE CRITERIA
==================================================

- muestra datos reales de todos los módulos
- no hay duplicación de datos
- prioridades correctas
- orden correcto
- dashboard usable sin navegación adicional
- sin impacto negativo en performance

==================================================
VALIDATIONS REQUIRED
==================================================

- php lint
- carga dashboard sin error
- datos correctos por usuario
- no regresión en CRM / calendar / tasks

DOCS TO UPDATE
==================================================

- docs/PLUGIN_ROADMAP.md

==================================================
TECHNICAL DEBT
==================================================

- filtros avanzados
- paginación si escala
