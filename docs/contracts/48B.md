PHASE: 48B
NAME: Roles y vistas refinadas

==================================================
OBJETIVO
==================================================

Refinar la visibilidad de bloques y acciones según rol operativo,
para que cada usuario vea una experiencia más relevante y menos cargada.

==================================================
SCOPE
==================================================

- visibilidad de bloques por rol
- visibilidad de acciones por rol
- refinamiento del dashboard por rol
- reutilización de lógica existente

==================================================
OUT OF SCOPE
==================================================

- nuevo sistema de roles
- cambios de capabilities base
- dashboards completamente distintos
- cambios de lógica de negocio
- CRM Pipeline
- personalización por usuario individual

==================================================
ALLOWED FILES
==================================================
- includes/dashboard/class-admin-dashboard-controller.php
- includes/dashboard/class-workload-service.php
- includes/config/class-operational-config-service.php si hace falta lectura de flags por rol
- docs/contracts/48B.md
- docs/contracts/validation/48B-validation.md

==================================================
CRITERIO DE IMPLEMENTACIÓN
==================================================
- no duplicar bloques
- no duplicar lógica
- usar rol/capability actual
- ocultar lo irrelevante, no recalcular distinto
- mantener fallback seguro para admin

==================================================
OBJETIVO FUNCIONAL
==================================================

Ejemplos esperados:

- admin:
  - ve dashboard completo operativo
  - ve automation summary
  - ve métricas globales

- mechanic:
  - ve foco operativo
  - ve mi trabajo
  - ve quick actions relevantes
  - menos bloque administrativo/técnico

- client:
  - no debe ver dashboard interno administrativo
  - solo vistas que ya le correspondan por portal/shortcodes

==================================================
ACCEPTANCE CRITERIA
==================================================

- dashboard más relevante por rol
- menos ruido para mecánico
- admin conserva visibilidad necesaria
- cliente no expuesto a capas internas
- sin regresión funcional
