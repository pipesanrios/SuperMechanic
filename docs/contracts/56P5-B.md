# TASK CONTRACT — 56P5-B Cascade delete

## Objective
Ensure CRM opportunity deletion safely cascades to related CRM tasks so no orphaned operational tasks remain after delete actions.

## Scope
- CRM delete cascade only
- related task cleanup tied to opportunity deletion
- safe deletion consistency

## In Scope
- delete single opportunity -> delete related tasks
- delete bulk-selected opportunities -> delete related tasks
- preserve safe delete flow
- avoid orphan CRM task records

## Out of Scope
- CRM redesign
- task model redesign
- API
- i18n
- reset logic
- schema changes unless strictly unavoidable

## Allowed Files
- includes/crm/*
- includes/database/* only if strictly needed for CRM delete consistency
- docs/CURRENT_STATE.md
- docs/QA_REPORT.md

## Forbidden Files
- includes/users/*
- includes/modules/*
- REST/API files
- process/payment files
- admin menu/settings files unless strictly needed for notices only

## Constraints
- minimal scoped changes
- preserve existing CRM behavior
- no broad redesign
- safe delete only

## Deliverables
- deleting CRM opportunity also removes related CRM tasks
- no orphan CRM tasks remain from delete flow
- docs updated

## Validation
- php scripts/php-lint.php --all
- manual check:
  - single delete cascades to tasks
  - bulk delete cascades to tasks
  - no admin regression
