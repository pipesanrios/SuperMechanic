PHASE: 50F
NAME: Webhooks Admin UI

==================================================
OBJETIVO
==================================================

Crear una interfaz administrativa simple para gestionar webhooks
desde WordPress Admin, usando la infraestructura ya creada en 50D.

==================================================
SCOPE
==================================================
- página/menu de webhooks
- listado de webhooks
- crear webhook
- editar webhook
- activar/desactivar webhook
- eliminar webhook
- test manual de webhook

==================================================
OUT OF SCOPE
==================================================
- editor avanzado de payloads
- logs avanzados de webhook
- retries / queues
- webhooks condicionales complejos
- CRM Pipeline

==================================================
ALLOWED FILES
==================================================
- includes/webhooks/class-webhook-repository.php
- includes/webhooks/class-webhook-service.php
- includes/admin/class-webhooks-admin-controller.php (nuevo)
- assets/css/admin.css
- assets/js/admin-webhooks.js (nuevo si hace falta)
- docs/contracts/50F.md
- docs/contracts/validation/50F-validation.md

==================================================
CRITERIO DE IMPLEMENTACIÓN
==================================================
- reutilizar backend existente
- no duplicar lógica de dispatch
- UI simple, clara y usable
- acciones seguras con nonce + capability
- mantener coherencia con panel admin actual

==================================================
FUNCIONALIDAD REQUERIDA
==================================================

1. Menú:
- Super Mechanic → Webhooks

2. Listado:
- name
- url
- event_type
- is_active
- secret_key (oculto/parcial)
- created_at
- updated_at

3. Acciones:
- Create
- Edit
- Activate / Deactivate
- Delete
- Send test

4. Formulario create/edit:
- name
- url
- event_type
- secret_key (opcional)
- active yes/no

5. Test webhook:
- enviar payload estándar de prueba
- no bloquear UI si falla
- mostrar success/error claro

==================================================
EVENTOS DISPONIBLES
==================================================
- membership_created
- membership_updated
- user_transferred
- overdue_alert_detected
- critical_signal_detected

==================================================
SEGURIDAD
==================================================
- capability: sm_manage_plugin
- nonce obligatorio
- sanitización estricta
- validar URL
- validar event_type soportado

==================================================
ACCEPTANCE CRITERIA
==================================================
- página visible y usable
- CRUD básico de webhooks funciona
- test webhook funciona
- dispatch base no se rompe
- sin regresión en notifications/webhooks

==================================================
VALIDATIONS REQUIRED
==================================================
- php lint
- QA runner
- runtime manual
