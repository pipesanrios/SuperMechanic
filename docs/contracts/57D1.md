## Acceptance Criteria
- valid jobs can be built for:
  - `inventory_import`
  - `inventory_connector_sync`
  - `email_delivery`
  - `webhook_delivery`
  - `google_calendar_sync`
  - `pdf_generation`
- valid jobs expose normalized fields:
  - `job_id`
  - `job_type`
  - `business_id`
  - `tenant_id`
  - `payload`
  - `status`
  - `attempts`
  - `max_attempts`
  - `scheduled_at`
  - `created_at`
  - `last_error`
- invalid jobs fail for:
  - missing `job_type`
  - invalid `job_type`
  - missing `business_id`
  - invalid `payload`
  - `attempts > max_attempts`
- dispatcher returns passive queue results with:
  - `writes = 0`
  - `executed = false`
  - `passive = true`
- status model exposes:
  - `pending`
  - `running`
  - `completed`
  - `failed`
  - `retry_scheduled`
  - `cancelled`
- no persistence, workers, cron, external calls, schema changes, connector execution, email sending or PDF generation occurs.
