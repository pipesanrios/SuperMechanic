## Objective
Implement a safe CSV import foundation for Mekvort vehicle catalog records.

## Scope

- validation
- dry-run/preview
- controlled import into vehicle catalog

## In Scope
- CSV parser/service
- required column validation

- import execution
- create/update catalog records

## Out of Scope

- scheduled sync
- API import

## Constraints
- Controller -> Service -> Repository -> Database
- no SQL outside repository

- import must be business-scoped

## Validation Contract
- `docs/contracts/validation/56P12-D-validation.md`

## Acceptance Criteria
- CSV import accepts required columns:
  - `make`
  - `model`
  - `year`
- CSV import accepts optional columns:
  - `trim_version`
  - `body_type`
  - `fuel_type`
  - `transmission`
  - `engine`
  - `notes`
  - `status`
- Dry-run validates headers and rows without writing to DB.
- Dry-run reports total rows, valid rows, invalid rows, row errors and preview rows.
- Confirmed import writes valid rows into `sm_vehicle_catalog`.
- Import uses `Vehicle_Catalog_Service`.
- Imported records require and preserve business scope.
- Admin import actions enforce capability, nonce and CSV file validation.
- No external connector, scheduled sync, VIN decoder, CRM/reset/API change or customer vehicle creation is added.

## Deliverables

- import execution
- admin entrypoint for upload/preview/import if needed
