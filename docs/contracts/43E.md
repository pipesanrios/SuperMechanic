PHASE: 43E
NAME: Motor de reglas persistente

==================================================
OBJETIVO
==================================================

Permitir que reglas operativas sean configurables y persistentes
por business_id, sin depender de valores hardcodeados.

==================================================
SCOPE
==================================================

- persistencia de reglas en DB
- lectura de configuración por tenant
- fallback a defaults si no existe configuración
- integración con evaluación existente (42E–43D)

==================================================
OUT OF SCOPE
==================================================

- UI avanzada de edición
- cron
- DSL compleja
- versionado histórico
- auditoría avanzada
- tocar CRM Pipeline core

==================================================
ALLOWED FILES
==================================================

- includes/automation/class-operational-rules-service.php
- includes/automation/class-operational-rules-repository.php (nuevo)
- includes/automation/class-operational-rules-installer.php (nuevo)
- includes/dashboard/class-workload-service.php
- includes/dashboard/class-admin-dashboard-controller.php
- docs/contracts/43E.md
- docs/contracts/validation/43E-validation.md

==================================================
OBJETIVO FUNCIONAL
==================================================

- cada regla debe poder:
  - activarse/desactivarse
  - cambiar modo ejecución (manual/confirmable/auto)
  - ajustar thresholds básicos
  - ajustar límites (ej: max items auto)

==================================================
CRITERIO DE IMPLEMENTACIÓN
==================================================

- repository pattern obligatorio
- SQL SOLO en repository
- service consume repository
- fallback a defaults si no hay fila
- no romper reglas actuales

==================================================
ESTRUCTURA DE REGLA (OUTPUT)
==================================================

```php
[
  'rule_key' => 'overdue_tasks_cleanup',
  'enabled' => true,
  'execution_mode' => 'auto|confirmable|manual',
  'thresholds' => [...],
  'limits' => [...],
  'source' => 'db|default'
]
