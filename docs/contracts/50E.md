PHASE: 50E
NAME: Advanced Automation Base

==================================================
OBJETIVO
==================================================

Implementar una capa base de automatización avanzada
para encadenar eventos, condiciones y acciones internas
de forma desacoplada y extensible.

==================================================
SCOPE
==================================================
- automation service central
- reglas simples evento -> condición -> acción
- ejecución desacoplada
- integración con notificaciones y webhooks ya existentes

==================================================
OUT OF SCOPE
==================================================
- editor visual de automatizaciones
- builder complejo de reglas
- colas avanzadas
- interfaz tipo Zapier
- CRM Pipeline invasivo

==================================================
ALLOWED FILES
==================================================
- includes/automation/class-automation-engine-service.php
- includes/automation/class-operational-rules-service.php
- includes/notifications/class-notification-service.php
- includes/webhooks/class-webhook-service.php
- docs/contracts/50E.md
- docs/contracts/validation/50E-validation.md

==================================================
CRITERIO DE IMPLEMENTACIÓN
==================================================
- no romper lógica actual
- no duplicar disparos existentes
- motor simple y legible
- acciones desacopladas
- extensible para futuras fases

==================================================
MODELO BASE
==================================================

Automation_Engine_Service

Métodos mínimos:
- handle_event($event_type, $payload = [])
- resolve_automation_actions($event_type, $payload = [])
- execute_automation_action($action, $payload = [])

==================================================
EVENTOS INICIALES
==================================================
- membership_created
- membership_updated
- user_transferred
- overdue_alert_detected
- critical_signal_detected

==================================================
ACCIONES INICIALES
==================================================
- send_notification
- dispatch_webhook
- create_internal_flag (si existe punto limpio)
- add_operational_note (solo si existe punto limpio)

Si no existe punto limpio para flag/note:
- dejar preparado, pero no forzar integración

==================================================
ACCEPTANCE CRITERIA
==================================================
- engine responde a eventos soportados
- acciones se resuelven correctamente
- notificaciones/webhooks pueden ejecutarse desde engine
- sin regresión funcional
- sin duplicación evidente

VALIDATIONS REQUIRED
