PHASE: 43C
NAME: Ejecución automática controlada

==================================================
OBJETIVO
==================================================

Permitir ejecución automática controlada de acciones operativas
de bajo riesgo, bajo reglas estrictas y sin intervención humana,
pero con límites, trazabilidad y seguridad.

==================================================
SCOPE
==================================================

- ejecutar automáticamente acciones seguras ya existentes
- limitar alcance a casos de bajo riesgo
- registrar resultado visible en dashboard
- no introducir automatización agresiva

==================================================
OUT OF SCOPE
==================================================

- auto reasignación compleja
- auto ejecución sobre procesos
- auto ejecución sobre citas
- cron externo complejo
- nuevas tablas
- rollback automático aún
- tocar CRM Pipeline core

==================================================
ALLOWED FILES
==================================================

- includes/automation/class-operational-rules-service.php
- includes/dashboard/class-workload-service.php
- includes/dashboard/class-admin-dashboard-controller.php
- docs/contracts/43C.md
- docs/contracts/validation/43C-validation.md

==================================================
FORBIDDEN FILES
==================================================

- includes/modules/*
- schema/database
- includes/processes/*
- includes/appointments/*
- invoices/payments core

==================================================
OBJETIVO FUNCIONAL
==================================================

Permitir autoejecución controlada SOLO para reglas marcadas como auto-executable.

Primera regla soportada:

1. overdue_tasks_cleanup
   - auto ejecuta `bulk_resolve`
   - solo si:
     - grupo ejecutable existe
     - cantidad <= límite seguro
     - todos los items son `crm_task`
     - estado sigue siendo compatible

Reglas no autoejecutables en 43C:
- critical_saturation_rebalance
- multi_critical_alert

==================================================
CRITERIO DE IMPLEMENTACIÓN
==================================================

- reutilizar handlers ya existentes
- no duplicar lógica de ejecución
- no crear motor paralelo
- autoejecución solo por llamado explícito controlado
- no cron obligatorio en esta fase
- registrar resultado en capa visible del dashboard
