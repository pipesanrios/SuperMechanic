# TASK CONTRACT — 56P9-B Calendar sync logic

## Objective
Implement internal Google Calendar sync logic foundations by building normalized calendar event payloads from Mekvort domain data, without OAuth execution or external Google API calls.

## Scope
- calendar event payload builder
- sync service foundation
- mapping Mekvort appointments/processes to calendar-ready payloads

## In Scope
- Google_Calendar_Sync_Service
- build_event_payload()
- build_appointment_event_payload()
- build_process_event_payload()
- validation of required event fields
- debug/intent output only

## Out of Scope
- OAuth
- token exchange
- Google API calls
- real calendar event creation
- cron/queue
- admin UI redesign
- schema changes unless strictly unavoidable

## Allowed Files
- includes/services/*
- includes/helpers/*
- docs/CURRENT_STATE.md
- docs/QA_REPORT.md

## Forbidden Files
- frontend css/js
- CRM
- reset engine
- API layer
- dashboard controllers
- process/business logic files unless strictly required for read-only data mapping

## Constraints
- no external API calls
- no OAuth
- no real sync yet
- mapping only
- reusable for 56P9-C

## Deliverables
- Google_Calendar_Sync_Service
- normalized calendar payload builder
- appointment/process mapping support
- docs updated

## Acceptance Criteria
- `includes/services/class-google-calendar-sync-service.php` exists.
- `Google_Calendar_Sync_Service` builds normalized calendar-ready payloads.
- appointment payloads include `summary`, `description`, `start.datetime`, `end.datetime`, `timezone`, `attendees` and required metadata.
- process payloads include `summary`, `description`, `start.datetime`, `end.datetime`, `timezone`, `attendees` and required metadata.
- `validate_event_payload()` returns structured errors for missing required fields.
- no OAuth flow, token exchange, Google API calls, real calendar event creation, schema changes, frontend/API/CRM/reset changes are introduced.
