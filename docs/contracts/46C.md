PHASE: 46C
NAME: Lazy loading

==================================================
OBJETIVO
==================================================

Dividir la carga del dashboard para que los bloques prioritarios
se rendericen primero y los bloques pesados se carguen de forma diferida.

==================================================
SCOPE
==================================================

- lazy loading de bloques no críticos
- render inicial rápido
- endpoints/admin-ajax o equivalente seguro
- placeholders simples

==================================================
OUT OF SCOPE
==================================================

- SPA completa
- React/Vue
- cambios de lógica de negocio
- lazy loading de acciones críticas
- lazy loading de ejecución de reglas

==================================================
ALLOWED FILES
==================================================
- includes/dashboard/class-admin-dashboard-controller.php
- includes/dashboard/class-workload-service.php
- assets/js/admin-dashboard.js
- assets/css/admin.css
- docs/contracts/46C.md
- docs/contracts/validation/46C-validation.md

==================================================
CRITERIO DE IMPLEMENTACIÓN
==================================================

- render inicial debe priorizar operación
- bloques pesados deben cargar después
- usar endpoints seguros
- no romper UX si falla carga diferida
- placeholders claros

==================================================
ACCEPTANCE CRITERIA
==================================================

- dashboard se siente más rápido
- bloques pesados cargan después sin romper UI
- bloques críticos cargan primero
- sin regresión 43A–46B

==================================================
VALIDATIONS REQUIRED
==================================================
- php lint
- QA runner
- runtime manual

==================================================
TECHNICAL DEBT
==================================================
- sin skeleton avanzado
- sin batching sofisticado
- sin cancelación de requests
