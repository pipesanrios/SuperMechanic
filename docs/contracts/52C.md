PHASE: 52C
NAME: Advanced Logging Base

==================================================
OBJETIVO
==================================================

Implementar una capa de logging avanzado para registrar
eventos técnicos clave del sistema con contexto útil
para debugging y observabilidad.

==================================================
SCOPE
==================================================
- tabla de logs
- log service central
- registro estructurado
- integración con queue, notifications, webhooks y automation

==================================================
OUT OF SCOPE
==================================================
- viewer UI avanzado
- analytics
- dashboards de métricas
- exportación de logs
- alertas automáticas

==================================================
ALLOWED FILES
==================================================
- includes/logs/class-log-installer.php (nuevo)
- includes/logs/class-log-repository.php (nuevo)
- includes/logs/class-log-service.php (nuevo)
- includes/queue/class-queue-service.php
- includes/notifications/class-notification-service.php
- includes/webhooks/class-webhook-service.php
- includes/automation/class-automation-engine-service.php
- docs/contracts/52C.md
- docs/contracts/validation/52C-validation.md

==================================================
MODELO REQUERIDO
==================================================

Tabla: sm_logs

Campos mínimos:
- id
- log_type
- source
- reference_id
- status
- message
- context_json
- created_at

==================================================
LOG TYPES INICIALES
==================================================
- queue
- notification
- webhook
- automation

==================================================
STATUS INICIALES
==================================================
- info
- success
- warning
- error

==================================================
SERVICE REQUERIDO
==================================================

Log_Service

Métodos mínimos:
- create_log($log_type, $source, $status, $message, $context = [], $reference_id = 0)
- log_queue_event(...)
- log_notification_event(...)
- log_webhook_event(...)
- log_automation_event(...)

==================================================
CRITERIO DE IMPLEMENTACIÓN
==================================================
- payload resumido en context_json
- no loggear datos sensibles de forma excesiva
- no romper performance
- integración ligera y útil

==================================================
INTEGRACIÓN REQUERIDA
==================================================

Registrar al menos:
- enqueue / process / retry / failed en queue
- send / fail en notifications
- dispatch / fail / success en webhooks
- handle_event / action execution en automation

==================================================
ACCEPTANCE CRITERIA
==================================================
- logs se crean correctamente
- tipos/status correctos
- context útil para debugging
- sin regresión funcional

==================================================
VALIDATIONS REQUIRED
==================================================
- php lint
- QA runner
- runtime manual
