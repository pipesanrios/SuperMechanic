PHASE: 51B
NAME: Branding / White-label Base

==================================================
OBJETIVO
==================================================

Permitir personalizar la identidad visual básica del sistema
(nombre, logo y colores principales) para uso white-label,
sin modificar la lógica de negocio.

==================================================
SCOPE
==================================================
- configuración de branding
- service central de branding
- UI admin básica para branding
- aplicación visual en áreas clave del admin

==================================================
OUT OF SCOPE
==================================================
- builder completo de temas
- personalización avanzada por usuario
- CSS dinámico complejo
- branding frontend completo del portal cliente
- CRM Pipeline

==================================================
ALLOWED FILES
==================================================
- includes/branding/class-branding-service.php (nuevo)
- includes/admin/class-branding-admin-controller.php (nuevo)
- includes/class-plugin.php
- assets/css/admin.css
- docs/contracts/51B.md
- docs/contracts/validation/51B-validation.md

==================================================
CONFIGURACIÓN REQUERIDA
==================================================

Branding settings mínimos:
- system_name
- logo_url o logo_attachment_id
- primary_color
- secondary_color
- admin_footer_text (opcional)

Persistencia:
- WP options simple

==================================================
SERVICE REQUERIDO
==================================================

Branding_Service

Métodos mínimos:
- get_settings()
- get($key, $default = null)
- save_settings($data)
- get_system_name()
- get_primary_color()
- get_logo_url()

==================================================
CRITERIO DE IMPLEMENTACIÓN
==================================================
- defaults seguros si no hay branding configurado
- no romper la experiencia actual
- no reemplazar toda la UI todavía
- aplicar branding solo en puntos claros y seguros

==================================================
APLICACIÓN VISUAL INICIAL
==================================================

Aplicar branding al menos en:
- página principal del plugin (header/título)
- menú/títulos del admin del plugin
- páginas nuevas principales:
  - License
  - Notifications
  - Webhooks
  - Roles & Access

Opcional:
- footer interno del plugin
