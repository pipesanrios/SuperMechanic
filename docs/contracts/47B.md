PHASE: 47B
NAME: Reducción de ruido en reglas

==================================================
OBJETIVO
==================================================

Reducir redundancia entre flags, sugerencias, escalamiento y acciones
para que el sistema muestre menos ruido y más claridad operativa.

==================================================
SCOPE
==================================================

- detectar outputs redundantes
- consolidar mensajes equivalentes
- evitar repetición del mismo problema en múltiples bloques
- mantener una sola acción clara por problema cuando aplique

==================================================
OUT OF SCOPE
==================================================

- cambios de lógica de negocio
- cambios de reglas base
- cambios de ejecución automática
- CRM Pipeline
- scoring nuevo
- machine learning

==================================================
ALLOWED FILES
==================================================
- includes/dashboard/class-workload-service.php
- includes/dashboard/class-admin-dashboard-controller.php
- docs/contracts/47B.md
- docs/contracts/validation/47B-validation.md

==================================================
CRITERIO DE IMPLEMENTACIÓN
==================================================

- no eliminar información crítica
- no ocultar acciones ejecutables útiles
- preferir consolidación sobre duplicación
- mantener explicación clara y directa
- no romper performance

==================================================
OBJETIVO FUNCIONAL
==================================================

Cuando un mismo problema aparezca en varias capas (ej. overdue backlog):
- debe existir una salida prioritaria principal
- las demás salidas deben resumirse, degradarse o evitar duplicarse visualmente

Ejemplos típicos:
- overdue tasks
- delayed processes
- critical signals
- workload saturation

==================================================
ACCEPTANCE CRITERIA
==================================================

- dashboard muestra menos duplicidad
- una misma situación no domina múltiples bloques innecesariamente
- sigue habiendo claridad operativa
- sin regresión funcional
- acciones críticas siguen visibles
