PHASE: 42A
NAME: Acciones Operativas Asistidas

==================================================
OBJETIVO
==================================================

Permitir acciones manuales guiadas desde dashboard sobre
ítems operativos detectados por el sistema inteligente.

==================================================
SCOPE
==================================================

- acciones manuales desde dashboard
- ejecutar rutas seguras ya existentes
- usar sugerencias como punto de entrada
- no crear automatización autónoma

==================================================
OUT OF SCOPE
==================================================

- autoejecución de acciones
- nuevas tablas
- cron
- emails / WhatsApp / push
- cambios masivos
- reasignación real automática
- cambios destructivos sin confirmación
- tocar CRM Pipeline core

==================================================
ALLOWED FILES
==================================================

- includes/dashboard/class-workload-service.php
- includes/dashboard/class-admin-dashboard-controller.php
- assets/css/admin.css
- docs/contracts/42A.md
- docs/contracts/validation/42A-validation.md

==================================================
FORBIDDEN FILES
==================================================

- includes/modules/*
- schema/database
- includes/crm/* (salvo consumo indirecto ya existente)
- includes/processes/*
- includes/appointments/*
- invoices/payments core

==================================================
OBJETIVO FUNCIONAL
==================================================

Agregar acciones asistidas desde dashboard para sugerencias y flags operativos.

Ejemplos válidos:

1. abrir CRM con filtro relevante
2. abrir procesos activos filtrados
3. abrir citas próximas
4. abrir workload crítico del usuario
5. abrir área relacionada según `action_hint`

Las acciones deben ser:

- manuales
- seguras
- reversibles a nivel de navegación
- sin mutación directa de datos

==================================================
CRITERIO DE IMPLEMENTACIÓN
==================================================

- reutilizar 41C y 41E
- transformar recomendaciones en acciones navegables
- no duplicar lógica de destino
- no ejecutar cambios de estado
- no escribir en DB

==================================================
ESTRUCTURA DE SALIDA
==================================================

```php
[
  'actions' => [
    [
      'key' => 'open_overdue_tasks',
      'label' => 'Open overdue CRM tasks',
      'url' => '...',
      'level' => 'warning|critical',
      'context' => '...',
    ]
  ],
  'summary' => [
    'total' => int,
    'critical' => int,
    'warning' => int,
  ]
]
