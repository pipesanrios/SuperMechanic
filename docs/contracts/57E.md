# TASK CONTRACT - 57E Connector Runtime Wiring

## Objective
Wire the mock inventory connector prototype to the passive async queue foundation without executing real workers, real sync, imports, cron or external providers.

## Scope
- passive connector sync intent bridge
- passive dry-run intent bridge
- queue job payload normalization for connector runtime intent
- runtime smoke validation for mock connector -> passive queue job
- documentation updates

## In Scope
- `Inventory_Connector_Service` passive queue bridge methods
- `Queue_Dispatcher` usage for `inventory_connector_sync`
- dry-run and sync-simulation intent payloads
- mock connector local data only
- runtime smoke with no writes/execution
- current state, QA, roadmap and task docs

## Out of Scope
- real provider APIs
- OAuth
- scheduled sync
- real job execution
- queue persistence
- workers
- catalog import execution
- database writes
- schema changes
- admin UI
- API/frontend changes

## Allowed Files
- includes/integrations/inventory-connectors/*
- includes/saas/*
- docs/*

## Forbidden Files
- CRM
- users/roles
- process/payment logic
- API controllers
- frontend/assets
- database/schema
- external integrations outside inventory connector docs/code
- real provider APIs

## Constraints
- no external API calls
- no OAuth
- no scheduled sync
- no real job execution
- no DB writes
- no schema changes
- no admin UI
- preserve mock connector as local-only provider
- preserve passive queue dispatcher semantics

## Deliverables
- `Inventory_Connector_Service::build_sync_job(...)`
- `Inventory_Connector_Service::build_dry_run_job(...)`
- `Inventory_Connector_Service::dispatch_sync_intent(...)`
- `Inventory_Connector_Service::dispatch_dry_run_intent(...)`
- passive payload including:
  - `connector_key`
  - `operation`
  - `dry_run`
  - `provider_type`
  - normalized item preview/count
  - validation summary
- runtime smoke evidence
- docs updated

## Acceptance Criteria
- mock connector dry-run can build an `inventory_connector_sync` passive queue job.
- mock connector sync simulation can build an `inventory_connector_sync` passive queue job.
- queue result has:
  - `passive = true`
  - `writes = 0`
  - `executed = false`
- payload contains connector identity, operation, dry-run flag, provider type, normalized item preview/count and validation summary.
- no DB writes, real import, real sync, worker, cron, external API, OAuth, schema change or admin UI is introduced.

## Validations Required
- `php scripts/php-lint.php --all`
- `php scripts/qa-runner.php --contract=docs/contracts/validation/57E-validation.md --output=text`
- runtime smoke:
  - mock connector fetches/normalizes inventory
  - dry-run builds passive queue job
  - sync intent builds passive queue job
  - dispatcher returns passive queue result
- static/manual:
  - no `$wpdb` usage in new wiring
  - no cron hooks
  - no `wp_remote_*`
  - no real import writes
  - no active worker execution
  - no schema/database changes
