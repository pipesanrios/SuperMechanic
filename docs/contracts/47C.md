PHASE: 47C
NAME: Priorización automática de acciones

==================================================
OBJETIVO
==================================================

Priorizar automáticamente acciones y sugerencias operativas
según criticidad, impacto y urgencia, para mejorar usabilidad
y reducir ruido en operación diaria.

==================================================
SCOPE
==================================================

- scoring simple de prioridad
- orden automático de acciones
- orden automático de sugerencias
- orden automático de workload visible cuando aplique

==================================================
OUT OF SCOPE
==================================================

- machine learning
- scoring predictivo
- cambios de lógica de negocio
- cambios de ejecución automática
- cambios de reglas
- cambios de CRM Pipeline

==================================================
ALLOWED FILES
==================================================

==================================================
CRITERIO DE IMPLEMENTACIÓN
==================================================

- scoring simple, explícito y mantenible
- no cajas negras
- no duplicar lógica
- no cambiar resultados funcionales
- solo mejorar orden y prioridad visual

==================================================
MODELO DE PRIORIDAD
==================================================

La prioridad puede considerar:

- severity base:
  - critical > warning > normal
- impact:
  - cantidad afectada
- urgency:
  - overdue / delay / due date próxima
- execution readiness:
  - ejecutable > no ejecutable

Debe quedar documentado en código de forma clara.

==================================================
ACCEPTANCE CRITERIA
==================================================

- acciones quedan mejor ordenadas
- sugerencias quedan mejor ordenadas
- items críticos y ejecutables aparecen primero
- sin regresión funcional
- orden coherente y explicable
