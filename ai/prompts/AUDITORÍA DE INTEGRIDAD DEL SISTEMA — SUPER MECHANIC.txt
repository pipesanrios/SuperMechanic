AUDITORÍA DE INTEGRIDAD DEL SISTEMA — SUPER MECHANIC

Objetivo:
auditar integridad tecnica real del plugin y detectar desalineaciones codigo/docs/contextos/prompts.

==================================================
FUENTE DE VERDAD
==================================================

Prioridad obligatoria:
1) codigo real (`includes/*`, bootstrap, schema, migraciones)
2) estructura DB real (`class-schema.php`)
3) docs tecnicas
4) contextos IA
5) prompts

Si hay conflicto: manda el codigo real.

==================================================
ALCANCE MINIMO OBLIGATORIO
==================================================

Revisar:
- bootstrap (`super-mechanic.php`, `includes/autoloader.php`, `includes/class-plugin.php`)
- modulos activos reales (`appointments`, `attachments`, `automation`, `businesses`, `clients`, `communication`, `dashboard`, `flows`, `helpers`, `integrations`, `invoices`, `maintenance`, `paperwork`, `predelivery`, `processes`, `quotes`, `relations`, `reports`, `vehicles`)
- `includes/database/*`
- uso/no uso de `includes/modules/*`
- seguridad documental (`Document_Service`, `Download_Service`, ownership/tenant boundary)
- tenancy (`business_id`, `sm_businesses`, `Business_Context_Service`)
- API interna y API publica
- webhooks outbound
- Google Calendar (sync + webhook/watch)

==================================================
CHECKS DE ARQUITECTURA
==================================================

Confirmar:
- `Controller -> Service -> Repository` coherente
- SQL solo en repositories (excepto `includes/database/*`)
- no dependencia a legacy `includes/modules/*`
- boundaries de modulo respetados

Detectar:
- SQL fuera de lugar
- lógica de negocio indebida en controllers/shortcodes
- acoplamientos peligrosos

==================================================
CHECKS DE DOCUMENTACION
==================================================

Contrastar contra:
- `ARCHITECTURE.md`
- `docs/CURRENT_STATE.md`
- `docs/SYSTEM_MAP.md`
- `docs/MODULE_REGISTRY.md`
- `docs/DATABASE_MAP.md`
- `docs/TEST_SCENARIOS.md`
- `docs/PLUGIN_ROADMAP.md`
- `.vscode/AI_CONTEXT.md`
- `ai/context/*`
- prompts activos en `ai/prompts/*`

Detectar:
- docs obsoletas
- fases mal mapeadas
- schema/version desactualizados
- duplicacion innecesaria

==================================================
SALIDA ESPERADA
==================================================

1) estado consolidado real
2) desalineaciones detectadas
3) riesgos/deuda tecnica
4) correcciones documentales recomendadas
5) estado final:
- ESTABLE
- ESTABLE CON RIESGOS
- NECESITA CORRECCIONES
- INCONSISTENTE
