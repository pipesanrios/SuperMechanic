Objetivo:
Revisar si el estado actual del plugin mantiene integridad arquitectónica, modular y de base de datos, y detectar si alguna fase o cambio reciente rompió dependencias.

Si existe cualquier diferencia entre documentación y código, usar el código real como fuente de verdad y reportar la desalineación.

==================================================
ALCANCE DE LA AUDITORÍA
==================================================

Debes revisar:

1. integridad de arquitectura
2. integridad de módulos
3. integridad de base de datos
4. integridad de bootstrap
5. integridad de permisos
6. integridad de shortcodes
7. integridad de flujos de negocio
8. coherencia entre documentación y código real
9. integridad del sistema documental y PDFs

==================================================
1. REVISAR ARQUITECTURA
==================================================

Confirmar que se sigue respetando:

- Repository = acceso a BD
- Service = lógica de negocio
- Controller = UI admin
- Shortcodes = frontend cliente
- REST Controller solo cuando aplique

Detectar:

- SQL fuera de repository
- lógica de negocio metida en controllers
- duplicación fuerte de lógica entre services
- módulos con responsabilidades mezcladas
- uso accidental de includes/modules/* como si fuera arquitectura activa
